# gonki

Технічне резюме проєкту: Multiplayer Racing 2026
Multiplayer Racing 2026 — це мережева гоночна гра в реальному часі, розроблена на Python. Проєкт реалізує механіку змагання двох гравців на одній трасі з перевіркою колізій та синхронізацією стану через центральний сервер.

🛠 Технологічний стек
Мова програмування: Python 3.x

Графічний інтерфейс: customtkinter (сучасна обгортка над tkinter).

Мережева взаємодія: socket (TCP), threading (багатопотоковість), json (серіалізація даних).

Обробка зображень: Pillow (PIL) для ротації та рендерингу спрайтів машин.

Аудіо: pygame.mixer для фонової музики.

🏗 Архітектура та реалізація
1. Серверна логіка (server20.py)
Управління з'єднаннями: Сервер очікує підключення рівно двох клієнтів перед початком гри.

Ідентифікація: Кожному гравцеві присвоюється унікальний ID (0 або 1) при підключенні.

Синхронізація (Broadcasting): Сервер працює як ретранслятор: отримує координати та кут повороту від одного клієнта і миттєво розсилає оновлений стан усім учасникам.

Тригер старту: Після збору гравців сервер надсилає широкомовну команду START_GAME.

2. Клієнтська логіка (client20.py)
Механіка руху: Реалізовано дискретне переміщення (крок 15 пікселів) з автоматичним поворотом спрайта машини на 0°, 90°, 180° або 270° залежно від напрямку.

Система колізій:

Дорога: Використовується метод find_overlapping, який перевіряє, чи знаходиться точка координат гравця в межах об'єкта "road" (чорна траса).

Фініш: Перевірка перетину заданої координатної лінії внизу траси.

Асинхронність: Мережеві запити обробляються в окремому потоці (threading.Thread), щоб графічний інтерфейс не "замерзав" під час очікування даних.

🎮 Ігровий процес
Керування: Розділене за ID гравця. Гравець 0 використовує клавіші WASD, Гравець 1 — стрілочки клавіатури.

Зворотний відлік: Перед початком гонки реалізовано візуальний таймер (3-2-1-РУШ!).

Візуалізація: Використовуються кастомні зображення машин (car_green.png, car_yellow.png). У разі їх відсутності система автоматично перемикається на кольорові прямокутники (fail-safe mechanism).

🔍 Ключові технічні рішення
JSON-обмін: Дані передаються у форматі JSON (наприклад, {"x": 200, "y": 170, "angle": 90}), що забезпечує гнучкість при додаванні нових параметрів (наприклад, стан пошкоджень).

Валідація на стороні клієнта: Клієнт сам перевіряє можливість ходу (чи не виїхала машина за межі траси), перш ніж надсилати оновлені координати серверу.
